First is, obviously, this very website. Yeah, pretty cool. All that html+css is simply beautiful. No need to compile it or anything. Can just run it locally or, like moi, serve it from an nginx. You know. It's simple. I did play around and made every section load dynamically with a little HTMX magic. But otherwise - plain and simple.

Here's the reference to the repo.

Did you know? Github provides a static server for when your repo is named like 

{USER_NAME}.github.io
. Which makes this repo available at traumst.github.io without a dedicated backend.

The backend


Second, there's the backend. It is written in Rust at the moment. Why Rust? Well, why not? I mean I could write it in C and it would take as much time. I guess I just prefer to... suffer in a... modern way. Yes, that's it. To hell with seg faults! All hail the borrow checker! But seriously, I know many languages. Rust is different. It makes me think different. Which in turn makes me see new depths in programming. Isn't that good enough of a reason?

You can find the repo for the backend here.

Oh, and this repo also contains the Dockerfile as well as some other automation-related stuff.
